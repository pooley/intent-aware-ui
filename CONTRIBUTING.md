# Contributing

First of all, thank you for your interest in Intent-Aware UI.

This project is currently in a research phase.

The primary objective is to define a robust interaction model before building production-ready implementations.

---

# How to Contribute

Contributions are welcome in many forms:

- Discussion
- Research
- UX critiques
- Accessibility expertise
- Scientific references
- Prototype implementations
- Documentation improvements
- Visual demonstrations

At this stage, thoughtful discussions are often more valuable than code.

---

# Before Opening an Issue

Please check whether your question or idea is already covered in:

- README
- FAQ
- Specification
- Existing Issues
- Discussions

---

# Design Principles

When proposing changes, try to preserve the project's core principles.

Intent-Aware UI should always remain:

- Predictable
- Progressive
- Accessible
- User-controlled
- Non-intrusive

If a proposal compromises one of these principles, explain why.

---

# Pull Requests

A good Pull Request should:

- Solve one problem.
- Stay focused.
- Include documentation updates when needed.
- Explain the reasoning behind the proposal.
- Remain consistent with the specification.

---

# Research Contributions

Research references are highly appreciated.

When possible, include:

- academic papers
- usability studies
- accessibility research
- HCI literature
- prior art

Intent-Aware UI aims to build upon existing research rather than replace it.

---

# Reporting Ideas

When proposing a new concept, consider answering:

- What problem does it solve?
- Why is it better than existing approaches?
- Does it preserve user agency?
- Does it improve accessibility?
- Does it remain predictable?
- Could it create unexpected behaviors?

# Roadmap

See [docs/roadmap.md](docs/roadmap.md) for the project's current phase and direction, kept there rather than duplicated here so it doesn't drift out of date.

---

# Philosophy

Intent-Aware UI is not about making interfaces more animated.

It is about making interactions require less effort while preserving user control.

Every contribution should support that vision.
