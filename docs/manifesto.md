## Introduction

For decades, graphical user interfaces have relied on the same interaction model.

Move a pointer.

Reach a target.

Click.

This model has proven remarkably successful. It shaped personal computing, the web, mobile devices and countless digital experiences.

Yet while our software has dramatically evolved, our primary interaction model has barely changed.

Today's interfaces still expect users to be precise.

Perhaps it is time for interfaces to become more understanding instead.

---

# Precision has become the hidden cost of modern interfaces

Software is becoming increasingly powerful.

Unfortunately, it is also becoming increasingly dense.

Modern interfaces contain:

- floating toolbars
- contextual actions
- compact menus
- icon-only controls
- secondary actions
- split buttons
- nested interactions

Every pixel now serves multiple purposes.

Visual efficiency often comes at the expense of motor precision.

Users are expected to perform increasingly accurate movements to accomplish increasingly simple tasks.

The burden remains entirely on the human.

---

# We optimized interfaces for screens

Responsive Design adapted interfaces to screen sizes.

Dark Mode adapted interfaces to lighting conditions.

Reduced Motion adapted interfaces to vestibular disorders.

Intent-Aware UI asks a different question:

**Why don't interfaces adapt to human motor abilities?**

---

# Human capabilities are not constant

No user is perfectly precise.

Motor precision constantly changes depending on context.

It is affected by:

- age
- fatigue
- stress
- illness
- injury
- input device
- environment

A desktop mouse behaves differently from a laptop trackpad.

A rested hand behaves differently from a tired one.

The same person behaves differently throughout the day.

Yet interfaces assume a constant level of precision.

Humans adapt.

Interfaces rarely do.

---

# Accessibility should not be exceptional

Motor impairments are often addressed through dedicated accessibility features.

These are essential.

But accessibility should not only exist as an optional mode.

An interface that naturally requires less precision benefits everyone.

Accessibility should emerge from better interaction design, not only from additional settings.

---

# The limits of target acquisition

Human-computer interaction has long relied on principles such as Fitts' Law.

Fitts demonstrated that larger targets and shorter distances reduce acquisition time.

These principles remain fundamental.

But today's interfaces increasingly struggle to apply them.

As software grows more complex:

- controls become smaller
- layouts become denser
- available space becomes limited

Simply making everything larger is no longer always possible.

Improving interaction may require changing behavior instead of geometry.

---

# The click is only the confirmation

Current interfaces understand only one certainty:

The click.

Everything before that moment is ignored.

The user's trajectory.

Their hesitation.

Their intention.

Their confidence.

None of it exists in the interaction model.

Yet people rarely move randomly.

Long before clicking, they are already expressing intent.

The click merely confirms it.

---

# From reactive interfaces to aware interfaces

Today's interfaces react after the interaction.

Intent-Aware UI proposes something different.

An interface that progressively understands what the user is trying to accomplish.

Not to decide for them.

Not to automate actions.

Not to remove control.

Only to reduce unnecessary effort.

---

# Assistance should never replace agency

Helping users should never mean taking control away.

An intent-aware interface remains predictable.

It never surprises.

It never guesses silently.

It never acts without confirmation.

Instead, it progressively adapts to the confidence of the interaction while leaving the final decision entirely to the user.

The click remains the confirmation.

The user remains in control.

---

# A new interaction paradigm

Intent-Aware UI is not a button animation.

It is not enlarged hitboxes.

It is a proposal.

A proposal that interfaces should stop considering interaction as a binary event.

And start considering it as a continuous expression of human intent.

---

## We believe

We believe interfaces should adapt to people.

Not the opposite.

We believe accessibility can become intrinsic instead of exceptional.

We believe interaction should reduce effort rather than demand precision.

We believe intent is part of the interaction.

And we believe the future of user interfaces is not only responsive.

It is aware.
