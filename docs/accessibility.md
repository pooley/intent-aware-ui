# Accessibility

> Accessibility is not about exceptional users. It is about the natural diversity of human abilities.
>
> **Status:** Draft · **Version:** 0.1.0 · **Last Updated:** 2026-07-16

**[📚 Documentation Index](README.md)** · ← [Design Principles](design-principles.md) · [FAQ](faq.md) →

Intent-Aware UI explores interaction techniques that reduce the motor precision required to interact with digital interfaces.

Rather than targeting a specific group of users, the project is based on a simple observation:

Human motor abilities constantly change.

---

# Motor Variability

No one interacts with perfect precision all the time.

Motor performance naturally varies depending on many factors.

Examples include:

- age
- fatigue
- stress
- illness
- injury
- medication
- input device
- environment

The same person may experience different levels of precision throughout the same day.

Traditional interfaces rarely acknowledge this variability.

Intent-Aware UI explores whether interfaces themselves can adapt.

---

# Accessibility Benefits Everyone

Accessibility is often perceived as something designed for a minority of users.

In reality, accessible design improves usability for everyone.

Examples include:

- larger touch targets
- higher contrast
- keyboard navigation
- captions
- reduced motion

Intent-Aware UI follows the same philosophy.

Reducing unnecessary motor effort benefits all users.

---

# Accessibility Is Not a Mode

Many systems provide accessibility through optional settings.

These settings remain essential.

Intent-Aware UI does not replace them.

Instead, it explores whether interfaces can become naturally more forgiving without requiring users to explicitly enable assistance.

Accessibility should emerge from interaction design itself.

---

# Existing Accessibility Standards

Intent-Aware UI complements existing accessibility requirements.

It does not replace them.

Implementations should continue to follow:

- WCAG
- EN 301 549
- platform accessibility guidelines
- operating system accessibility APIs

The project assumes these foundations already exist.

---

# Motor Impairments

Some users may particularly benefit from reduced precision requirements.

Examples include people experiencing:

- Parkinson's disease
- Essential Tremor
- Cerebral Palsy
- Multiple Sclerosis
- arthritis
- repetitive strain injuries
- temporary injuries

Intent-Aware UI does not attempt to compensate for every impairment.

Instead, it investigates interaction models that may reduce acquisition effort while preserving user agency.

---

# Temporary Limitations

Permanent disabilities are only one source of motor variability.

Temporary situations are equally common.

Examples include:

- using a laptop on public transport
- carrying a child
- working with one hand
- using a damaged trackpad
- wearing gloves
- recovering from an injury

Designing for variability naturally improves resilience.

---

# User Agency

Accessibility should never come at the expense of control.

Intent-Aware UI follows one fundamental principle:

The interface may assist.

The user always decides.

Activation always requires explicit confirmation.

---

# Predictability

Assistance must remain understandable.

Users should never wonder why an interface moved or changed.

Visual feedback should explain the interaction rather than surprise the user.

Predictability is itself an accessibility feature.

---

# Accessibility by Design

Intent-Aware UI proposes that accessibility should not only consist of:

- larger buttons
- bigger spacing
- slower interactions

It may also emerge from adaptive interaction models.

This remains an open research question.

---

# Research, Not Replacement

Intent-Aware UI should be understood as a complementary approach.

The project does not replace existing accessibility techniques.

It explores an additional layer of interaction that may reduce effort while remaining compatible with established accessibility standards and best practices.

---

# Looking Forward

As interfaces continue to evolve, interaction models should evolve as well.

Accessibility is not only about removing barriers.

It is also about reducing unnecessary effort.

Intent-Aware UI explores one possible direction toward that goal.
